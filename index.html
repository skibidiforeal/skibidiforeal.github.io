Aurum Interpreter Documentation

This documentation covers the implementation of a basic interpreter for the language Aurum. The interpreter is implemented in C# and integrates with OpenAI's API to generate code based on user queries.

Introduction Key Components Functionality Breakdown Example Usage

Introduction

The Aurum interpreter is a simplified C#-based interpreter that supports basic programming constructs like arithmetic expressions, variable declarations, and conditional statements. It also features integration with OpenAI's GPT-3 model to generate code dynamically based on user-provided queries.

Features

  • Basic C#-like syntax for variable declarations, arithmetic operations, and print statements.
  • Conditional statements (e.g., if blocks).
  • Integration with OpenAI for AI-powered code generation based on user input.
  • Recursive handling of blocks and basic loops (to be implemented).

Key Components

1. Token and TokenType Classes

The TokenType enum represents different types of tokens in the language, such as numbers, arithmetic operators, identifiers, and AI-related commands. The Token class encapsulates the token type and the corresponding lexeme (text).

public enum TokenType { NUMBER, PLUS, MINUS, STAR, SLASH, ASSIGN, SEMICOLON, LEFT_BRACE, RIGHT_BRACE, IF, PRINT, IDENTIFIER, INT, EOF, AI }

public class Token
{
    public TokenType Type { get; }
    public string Lexeme { get; }

    public Token(TokenType type, string lexeme)
    {
        Type = type;
        Lexeme = lexeme;
    }

    public override string ToString()
    {
        return $"{Type} -> {Lexeme}";
    }
}

4. OpenAI API Integration

The HandleAICommand and QueryOpenAIForCode functions handle the integration with OpenAI's GPT-3 API. The user can provide a query, and the code will use OpenAI to generate code based on that query.

Functionality Breakdown

1. Variable Declarations

The interpreter supports variable declarations in C#-like syntax using int. For example:

int x = 5;
print x;

2. Arithmetic Operations

Basic arithmetic operators such as +, -, *, and / are recognized, although full expression evaluation is a future enhancement.

3. AI-Generated Code

The user can generate code dynamically using the ai = (query) command. This queries OpenAI and returns generated code based on the provided input.
